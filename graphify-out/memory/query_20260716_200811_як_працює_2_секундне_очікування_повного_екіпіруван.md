# Q: Як працює 2-секундне очікування повного екіпірування GearScore?

## Answer

Клієнт обмежує серверні запити інтервалом 2 секунди, повторно перевіряє GetItemInfo кожні 0.5 секунди й не завершує ServerPendingBuild, доки не доступні всі предмети. GearScore_ShouldWaitForServerRecord утримує екран Завантаження навіть для власного персонажа, тому частковий GearScore не показується. Сервер також має 2-секундний throttle, а offline SQL читає лише 18 екіпірованих слотів.
